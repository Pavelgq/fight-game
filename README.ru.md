# Fight Game

[English](README.md) · **Русский**

Прототип пошагового карточного файтинга на **Phaser 3** и **TypeScript**.  
Раскладываешь приёмы и шаги на таймлайне, выбираешь клетки на сетке 3×3 — раунд симулируется и проигрывается.

> Проект в активной разработке. Багрепорты и pull request'ы приветствуются.

## Статус и roadmap

**Сейчас:** MVP только на клиенте. ИИ-соперник и все правила выполняются в браузере (`src/model/`); бэкенда и постоянного хранения данных (кроме локального состояния сессии) нет.

**Планируется:**

- Серверная авторитетная логика мультиплеера (перенос разрешения раунда с клиента на доверенный сервер)
- Постоянные профили игроков/прогресс (сейчас только в памяти, см. `src/session/GameSession.ts`)
- Рост контента: больше приёмов, соперников и оставшиеся экраны (см. [открытые issues](https://github.com/Pavelgq/fight-game/issues))

Серверная часть, скорее всего, появится как отдельный пакет (например `packages/client` + `packages/server`), когда работа над ней реально начнётся — монорепо-каркас пока намеренно не заводим: делить между пакетами пока нечего.

## Возможности

- **Бой по таймлайну** — приёмы и смена стойки в рамках бюджета времени раунда
- **Стойки и дальность** — лево / центр / право; попадание зависит от столбца и `reach`
- **Зоны 3×3** — голова / корпус / ноги × лево / центр / право
- **Чистая модель** — правила в `src/model/`, тесты на Vitest, без Phaser
- **Слои UI** — изолированные views, layouts, screens и тонкие Phaser-сцены
- **Design 1280×720** — `Scale.FIT` и HiDPI-буфер canvas

## Геймдизайн

Два бойца не дерутся в реальном времени, а **планируют** ходы. Раунд устроен так:

1. **Планирование** — ты собираешь таймлайн из карт-приёмов и шагов смены стойки (лево/центр/право) в рамках бюджета времени на раунд; ИИ-соперник делает то же самое.
2. **Симуляция** — движок разрешает оба таймлайна по времени: попадания зависят от стойки, дальности (`reach`) и сетки 3×3 (голова/корпус/ноги × лево/центр/право); блоки, уклоны и сбивы могут отменить удар.
3. **Проигрывание** — оба таймлайна показываются рядом с маркерами попаданий и уроном.
4. **Завершение** — обновляется HP, стойки сбрасываются, добираются новые карты, начинается следующий раунд.

По ощущениям это смесь файтинга (дистанция, дальность, читка соперника) и карточной игры (рука, ресурсы), но разрешается детерминированно, а не в реальном времени.

Полные правила: [docs/GAME_DESIGN.ru.md](docs/GAME_DESIGN.ru.md) · Баланс (для мейнтейнеров): [.cursor/rules/battle-mechanics.mdc](.cursor/rules/battle-mechanics.mdc)

## Быстрый старт

**Нужно:** Node.js 22.12+ ([`.nvmrc`](.nvmrc))

```bash
git clone https://github.com/Pavelgq/fight-game.git
cd fight-game
npm install
npm run develop
```

Открой [http://localhost:4000](http://localhost:4000).

### Полезные URL для разработки

| URL | Назначение |
|-----|------------|
| `/?scene=BattleScene` | Сразу экран боя (фикстуры, если нет профиля) |
| `/?scene=BattleScene&debug=layout` | Бой с отладочной разметкой layout |

### Скрипты

| Команда | Описание |
|---------|----------|
| `npm run develop` | Dev-сервер на порту **4000** |
| `npm run build` | Сборка в `dist/` |
| `npm test` | Юнит-тесты Vitest |
| `npm run typecheck` | Проверка TypeScript |

## Структура проекта

```
src/
├── model/          # Правила, симуляция боя, способности (без Phaser)
├── session/        # GameSession — профиль и состояние матча
├── views/          # UI-компоненты (Button, AbilityCard, BattleGrid…)
├── screens/        # Сборка экранов (например BattleScreen)
├── controllers/    # Связка ввода и модели (BattleFieldController)
├── scenes/         # Phaser-сцены — только lifecycle и навигация
├── layouts/        # Координаты и размеры в design-пикселях (1280×720)
└── ui/             # Design system, масштаб, dev-фикстуры
```

Подробнее: [docs/ARCHITECTURE.ru.md](docs/ARCHITECTURE.ru.md)
