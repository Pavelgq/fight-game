# Обзор геймдизайна

[English](GAME_DESIGN.en.md) · **Русский**

Краткие правила пошагового боя. Константы баланса — `src/model/Battle/constants.ts`, полная спецификация — [`.cursor/rules/battle-mechanics.mdc`](../.cursor/rules/battle-mechanics.mdc).

## Цикл раунда

1. **Планирование** — оба бойца получают руку карт. Каждый собирает **таймлайн** приёмов и шагов в рамках **бюджета времени** раунда.
2. **Готово** — игрок подтверждает, ИИ планирует свой таймлайн, движок **симулирует** раунд.
3. **Проигрывание** — `RoundPlayback` показывает шкалы, удары, урон и сбивы.
4. **Итог** — HP, проверка нокаута, новый добор, сброс стоек в центр, усталость (если включена).

## Стойки (вместо дистанции)

Столбец стойки: **лево (0) / центр (1) / право (2)**.

- **Шаги** на таймлайне сдвигают стойку на ±1 (с зажимом).
- Атака должна **доставать** столбец соперника (`reach`: 0 — только свой столбец, 1 — соседний, 2 — любой).
- Попадание — если соперник в целевом столбце в момент удара; иначе **промах**.

## Сетка 3×3

| Строки | Зона |
|--------|------|
| 0 | Голова |
| 1 | Корпус |
| 2 | Ноги |

| Столбцы | Позиция |
|---------|---------|
| 0 | Назад (лево) |
| 1 | Центр |
| 2 | Вперёд (право) |

- **Поле игрока** (слева в UI) — **защита** на своей сетке.
- **Поле соперника** (справа) — **атаки** по клеткам соперника.

## Типы приёмов

| Тип | Куда ставится | Эффект |
|-----|---------------|--------|
| **Атака** | Клетка соперника | Урон при совпадении стойки и reach; блок / уклон |
| **Блок** | Своя клетка | Снижает урон по клетке, пока активен |
| **Уклон** | Без выбора клетки | Неуязвимость в зонах на время приёма |

У каждого приёма есть **скорость** (длительность на таймлайне). Длинные приёмы тратят больше бюджета.

## Порядок симуляции

События по **времени окончания** на таймлайне. Для каждой атаки:

1. Пропуск, если приём **сбит** более быстрым ударом
2. Стойки на момент удара
3. Reach и столбец → промах или дальше
4. Блок / уклон
5. Урон через `DamageCalculator`

Шаги меняют стойку по расписанию и влияют на последующие удары в том же раунде.

## Колода и рука

- Колода из списка приёмов бойца (`buildDeck`)
- Размер руки и добор — `battleConfig` в `constants.ts`
- Карта недоступна, если её speed больше **оставшегося бюджета** таймлайна

## ИИ

`planEnemyTimeline()` в `src/model/Battle/ai.ts` — эвристический планировщик. Можно заменить для сложности или PvP.
